# Keywords and Search Intent

Intent decides the page's *format*; keywords decide its *wording and placement*. Get intent right first — it outranks every tactic — then place keywords without stuffing.

## The four search intents

| Intent | Query looks like | Page format that ranks |
| --- | --- | --- |
| Informational | "how does X work", "what is X", "X guide" | Guide, explainer, definition, how-to |
| Commercial | "best X", "X vs Y", "X review", "X alternatives" | Comparison, listicle, evaluation |
| Transactional | "buy X", "X pricing", "X free trial", "X near me" | Product, pricing, signup, local landing |
| Navigational | "[brand] login", "[brand] docs" | The specific destination page |

**Confirm intent by looking at what already ranks.** Search the query and note the format of the top results. If they are all comparisons and you are writing a sales page, your format is wrong — fix that before any keyword work. The SERP is the answer key for intent.

## Choosing the primary keyword

- **One primary per page.** It should describe the page's single core topic and match a real query people search.
- Prefer a query you can realistically rank for over the highest-volume term — relevance and specificity beat raw volume, especially for newer sites (a "best X for [specific segment]" often beats the generic "best X").
- The primary keyword is what goes in the title, H1, first 100 words, and URL.

## Supporting and semantic terms

Modern engines rank on topical coverage, not single-keyword matching. Around the primary, cover:

- **Supporting keywords:** close variations and sub-queries the page should also answer.
- **Semantic / related terms:** the concepts and entities a thorough page on this topic would naturally mention. (Engines expect a page about "espresso machines" to also discuss "portafilter", "bar pressure", "milk frothing".)
- **Questions:** the real questions searchers ask, ideally as H2s or an FAQ.

Source these from the SERP (related searches, "people also ask"), the top-ranking pages' subheadings, and obvious sub-questions. Cover them because they make the page genuinely complete — not to hit a count.

## The keyword placement map

Place the primary keyword in these positions, each once, naturally:

| Position | Note |
| --- | --- |
| Title tag | Near the front |
| H1 | Matches the query |
| First 100 words | Confirms topic early |
| At least one subheading (H2/H3) | Where it reads naturally |
| URL slug | Short, hyphenated |
| Image alt text | Only where it honestly describes the image |
| Naturally through the body | As many times as normal writing requires — no target number |

Supporting and semantic terms go in subheadings and body where they fit the meaning.

## Over-optimization limits

- **There is no "keyword density target."** Write naturally; if a keyword appears awkwardly often, you have overdone it.
- **Exact-match repetition** ("best CRM software" five times verbatim) reads as manipulation and can hurt. Use variations and pronouns like a human would.
- **Hidden text, stuffed alt text, and keyword-jammed footers** are old tricks that now carry risk and no benefit.
- The test: read the page aloud. If the keyword usage sounds unnatural to a person, search engines treat it the same way. This is also exactly where the GEO pass (`ai-seo`) takes over — natural, extractable prose serves both ranking and citation.
